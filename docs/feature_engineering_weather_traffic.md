# Feature Engineering, Weather, and Traffic Signals

This note documents the engineered signals used by the Streamlit app and the current treatment of weather and traffic/corridor risk.

## Core Feature Engineering

The app joins procurement rows to the anonymized coordinate workbook using:

- `ShipToCode` for origin coordinates.
- `PurchaseName` for supplier destination coordinates.

After both coordinate matches are available, the app creates:

- `distance_km` using a vectorized Haversine calculation.
- `distance_category` as short, medium, or long.
- `late_days` from `ReceiptDate - RequiredDate`.
- `is_late` as a binary classification target.
- `delay_category` for delay severity grouping.
- `Backordered_qty`, `DeliveryStatus`, and `fulfillment_rate`.
- `order_value`, `item_order_frequency`, and `supplier_item_frequency`.
- `order_month`, `order_day_of_week`, and `lead_time_days`.
- `historical_item_avg_price`, `price_variance`, `price_ratio`, and `supplier_price_rank`.
- Supplier risk features including `late_rate`, `extension_rate`, `lateness_severity_score`, and `delivery_risk_score`.

## Weather Signal

The deployed Streamlit version uses a portable weather proxy named `weather_risk_index`.

Current logic:

- `high` for routes over 1,000 km.
- `medium` for routes over 300 km.
- `low` for shorter routes.
- `unknown` when distance is missing.

This avoids requiring API keys or live network access during deployment. The pipeline diagram notes the intended future enrichment path: query historical weather by origin coordinates and order dates, then replace the proxy with observed precipitation, wind, or storm-risk values.

## Traffic And Corridor Signal

The app uses two lightweight transport features:

- `transport_region`: local/domestic vs interstate classification based on distance.
- `transport_corridor_risk`: route exposure proxy based on distance.

Current logic:

- `high` corridor risk for routes over 1,000 km.
- `medium` for routes over 300 km.
- `low` for shorter routes.
- `unknown` when distance is missing.

This keeps the app portable for Streamlit Cloud. A production version could replace this with traffic data, road-network routing, or a spatial join against major freight corridors.
